断面で理解する: HBMはなぜ「積む」のか

HBMはDRAMコアダイをTSVで垂直貫通させ多段積層し、最下層のベースロジックダイに集約する。広帯域の源泉は2048bitの広いインタフェースを短い垂直配線で実現する点にある。16層への高積層化で熱密度とTSV整列精度が律速になり、接合はMR-MUFからハイブリッドボンディングへ移行していく。

系統B: ベースダイ・カスタムHBM

L4で最も投資的に面白い変化点。HBMが受動的記憶からロジックを抱えた共プロセッサへ変質する。ファウンドリ(TSMC)がメモリ価値連鎖に食い込む構造変化。

HBMベースダイ / カスタムHBM HBM Logic Base Die / C-HBM4E

ランプ/認定

HBMスタック最下層のロジックダイ。HBM4でDRAMプロセスでなくTSMC等の先端ロジックノードで製造され、HBMを「受動的記憶」から「能動的共プロセッサ」へ変える。C-HBM4Eで顧客がメモリロジックをファウンドリと共設計——L2(プロセス)・L5(GPU)とL4を結ぶ結節点。

系統C: メモリ階層拡張 (NAND / CXL)

HBMの外側を埋める階層。推論の長コンテキスト化でKVキャッシュ/RAGがHBM容量を超え、その溢れをSOCAMM2・エンタープライズSSD(NAND)・CXLが受ける。さらにHBFがHBMとNANDの中間を埋めにくる。メモリは単層でなく帯域-容量-コストの連続体になった。
