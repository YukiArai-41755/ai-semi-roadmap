Cerebras (ウェハスケール)

Cerebras WSE-3 / CS-3 — 300mmウェハ丸ごと1チップ、decode最速

300mmウェハ(直径12インチ)を切らずに丸ごと1つの巨大プロセッサにする「ウェハスケール」の逆転発想。チップレット/パッケージング(L3)が「いかに小さく分割して効率良く繋ぐか」を追求するのと真逆で、ダイシング自体を廃しオンチップSRAMだけでメモリの壁を物理的に回避する。WSE-3は46,225 mm²(普通のレチクル800mm²の57倍)、4兆トランジスタ、90万コア、44GB on-chip SRAM、21 PB/sメモリ帯域(H100の~7000倍)。decode(autoregressive生成)はメモリ帯域律速のため、CerebrasはHBMを必要としない設計で「Llama 3.1 70B を 2,100 tok/s/user、Llama 4 Maverickを2,500 tok/s/user」と、DGX B200の2倍超・H200の8倍のdecodeスループットを叩き出す。これがリアルタイム推論市場での独自地位を支える。2025-12のOpenAI契約($10-20B / 750MW)、2026 H2のAWS導入、2026-05のIPO申請($26.6B valuation、後に$185でClass A pricing)で「単なる新興」から「業界の第三極」へ昇格しつつある。

アーキテクチャ要点 — 「メモリの壁を物理で解く」

GPUがHBM3e/HBM4を必要とする本質的理由は「ダイの中だけでは数十GB級のモデルが入らないから」。Cerebrasはダイをウェハ全体に拡げることで、(1) 44GBをon-chip SRAMで持つ、(2) コア間距離が短くfabric帯域が桁違いに大きい、(3) 結果として「重みもKV cacheもon-chipに収まるサブセットなら、HBMアクセスが理論ゼロ」になる。decodeはtoken毎に全パラメータをスキャンするため、これがCerebrasの絶対的優位の源泉。一方、(a) 学習・大規模長コンテキストでは外部MemoryXに頼り、(b) chipレベルで電力 ~23 kW / 23 kg、(c) コスト$2-3M / unit級と単価が桁違い、(d) prefill(プロンプト処理)はGPU/Trainiumに任せる「decode専業」の使い分けが現実解。

システムレベル — CS-3 + Bedrock + Condor Galaxy

CS-3はラックの1コンポーネントとして売られるが、実際の運用は(1) AWS Bedrock経由でTrainium / Cerebrasを使い分け、(2) UAEのG42 "Condor Galaxy"でCS-3を数百〜数千台クラスタ、(3) OpenAI 750MW dedicatedといった大規模デプロイで進む。Bedrock上で「prefill = Trainium、decode = WSE-3」のような非対称推論を実装するのが特徴的な使い方で、これがNVIDIA monolithic設計に対する代替の現実解。
