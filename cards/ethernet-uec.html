Ethernet / UEC

Ultra Ethernet Consortium 1.0 / AI-Optimized Ethernet

ランプ/認定

InfiniBand 一強の AI scale-out 市場に対し、ハイパースケーラ・スイッチASICベンダ・NIC ベンダが結束したオープン標準。UEC は単なる RoCEv2 改良ではなく、Software/Transport/Network/Link/Physical の全層を AI ワークロード向けに再構築する。Packet Delivery Sublayer (PDS) による パケットレベルマルチパス・順序外配置 (out-of-order delivery)・ハードウェア再送・プログラマブル輻輳制御により、従来の RoCEv2 の「単一パス・順序保証・PFC 依存」によるインキャスト/ヘッドオブラインブロッキング問題を解決する。UEC 1.0 仕様は2025年6月に公開、Broadcom Thor Ultra (800G UEC NIC、10万XPU+対応) が業界初のUEC準拠商品として2025末出荷。Ethernet陣営が性能で InfiniBand に追いつき、TCO とエコシステムで逆転を狙う。

UEC 1.0 の主要技術 — 何が新しいか

層機能旧 RoCEv2 / 標準 Ethernet との差
Softwarelibfabric/UCX 連携 API、Collective offload (SHARP相当)標準化されたAI集合通信API、ベンダ間互換
Transport (UET)Packet Delivery Sublayer (PDS)、Selective ACK、HW再送、輻輳通知 (CSIG)RoCEv2 は単一フローpath固定、UET はパケット粒度マルチパス
NetworkマルチパスECMP、Adaptive Routingspray (パケット分散)+順序外配置で利用率最大化
LinkLink Layer Retry (LLR)、Credit-Based Flow Control (CBFC) 任意PFC ロスレス依存を緩和、ホップ毎再送可
PhysicalIEEE 802.3df (800G)、802.3dj (1.6T 200G/lane)準拠標準IEEE PHYに準拠、新規物理層は提案せず
セキュリティUET-Sec (AES-GCM全パケット暗号化、AES-XTS鍵管理)RoCEv2 では別レイヤだったWire encryption を標準化
運用Telemetry、INT、Congestion Marking統一マルチベンダ環境で観測性を統一

確立事実 UEC 1.0 は2025年6月公開、2025年10月に 1.0.1 minor update。UEC 1.1 は2026年中の公開予定(in-network collective、scale-up連携強化)。UALink / UEC は scale-up / scale-out の役割分担で並行進化。

確立事実 Meta は10万XPU超のスケールでEthernet (RoCEv2→UEC)を選択、Microsoft は IB と Ethernet の併用方針(Spectrum-X含む)、Google は内製TPUクラスタで Ethernet ベースの Jupiter fabric、xAI Colossus は IB から段階的に Ethernet (Spectrum-X) を併用。推論 10万XPU超のクラスタでは IB が運用上のスケール限界に近く、Ethernet/UEC が事実上の標準に。NVIDIA も Spectrum-X(独自Ethernet) を強化し、IB と Ethernet の二正面戦略。
