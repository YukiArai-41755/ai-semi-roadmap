CXL メモリプーリング

Compute Express Link 3.x / 4.0 — Cache-Coherent Memory Fabric

ランプ/認定

CPU・GPU・メモリ・アクセラレータをキャッシュコヒーレントに接続し、メモリをラック横断でプール共有する規格。PCIeの物理層を共有しつつ、メモリセマンティクス(load/store)とコヒーレンスプロトコルを上位に乗せる。HBMの容量制約(GPUあたり288GB級でも数千トークンのKVキャッシュで枯渇)を補い、ラック全体で「記憶」を共有することでメモリの壁をシステムレベルで緩和する。LLM長コンテキスト推論のKVキャッシュ・MoEモデルパラメータ・特徴量データベース等が主用途。メモリ中心アーキテクチャ × システム統合トレンドの交差点で、Marvell/Astera Labs/Samsung/SK hynix/Micronがハードウェア層、Microsoft Azureが2025-11に世界初の商用CXLメモリデプロイを始めた。

確立事実 CXL 3.1のGFAM (Global Fabric Attached Memory)は従来のtree topologyから脱却し、ラック横断・複数ホスト×複数メモリプールのfabric型接続を可能にする。これがメモリプーリングをラックスケールに拡張するキー機能。CXL 4.0は2025末批准済みだが、PCIe 7.0シリコン待ちで商用は2027〜2028。

CXL デバイス分類 (Type)

Typeプロトコル典型デバイスAI用途
Type 1CXL.io + CXL.cacheSmartNIC、アクセラレータ(memoryless)キャッシュコヒーレントなDPU/NIC
Type 2CXL.io + CXL.cache + CXL.memGPU、FPGA、AIアクセラレータGPUがホストメモリにcoherent、HBM拡張
Type 3CXL.io + CXL.memメモリエクスパンダ / メモリプールKVキャッシュ容量拡張の主役

推論 AI scale-up領域で本命となるのはType 3。Type 2はNVLink/UALinkに役割を取られる(GPU間接続はscale-up専用ファブリックが圧倒的に速い)が、Type 3メモリプールは「HBM(超高帯域、容量小)と DDR/LPDDR(中帯域、容量大)の階層化」の中段を埋める。NVIDIAも HBM3e/HBM4 + SOCAMM2 LPDDR + CXLという階層を志向。

確立事実 CXLレイテンシは~250-500 ns(local DRAM の2-3倍)。これは「コアの計算待ちにならない範囲」で、KVキャッシュ・コールド層のモデル重み・特徴量データベース等に適合する。プロダクション例: Azure M-seriesでCXL Flat Memory Modeを使い、メモリ容量を最大2倍に拡張(コスト効率)。

AI推論との結合 — なぜ今CXLか

推論 LLM長コンテキスト推論(100k〜1Mトークン)では、KVキャッシュサイズが線形に増える。Llama 3 70Bでも128kトークンで~40GB/seq、これが多並列実行になればHBMは即座に枯渇する。HBM増設はコスト爆発(HBM4で$15-25/GB級)で、CXLメモリプールでKVキャッシュを階層化(hot=HBM、warm=CXL、cold=NVMe)するのが推論コスト最適化の主戦場。MoEモデル(GPT-MoE、Mixtral、DeepSeek-V3等)も活性化エキスパートを動的にスワップするためCXLが効く。
