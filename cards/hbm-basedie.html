HBMベースダイ / カスタムHBM

HBM Logic Base Die / Custom HBM (C-HBM4E)

ランプ/認定

HBMスタックの最下層に置かれるロジックダイ。HBM2/2E/3/3eまでDRAMサプライヤ自身がDRAMプロセス(20〜10nm級)で内製していたが、HBM4からTSMC等の先端ロジックノード(N12FFC/N5/N3)で製造されることが業界標準化した。これによりベースダイにPHY、メモリコントローラ、ECC、近接演算、暗号化、テスト機能を高密度に詰め込めるようになり、HBMが「受動的記憶」から「能動的メモリサブシステム」へ変質する。C-HBM4Eでは顧客がベースダイの論理設計を持ち込み、SK hynix/Samsung/MicronのDRAMダイと組合せて発注する形態へ進む。これは「メモリ業界の請負ファブレス化」と表現でき、L2(プロセス)・L4(メモリ)・L5(演算)の境界を溶かす結節点。TSMCにとっては新たな先端ノード収益柱、NVIDIA/AMDにとってはメモリ差別化の鍵、メモリ3社にとっては設計参入障壁の上昇を意味する。

サプライチェーンの構造 — 4社co-design

HBMベースダイのco-designには通常以下の4プレイヤーが関与する。

プレイヤー担当
NVIDIA / AMD / AWS / Google等(顧客)必要なメモリSpec(帯域・容量・カスタム機能)を定義、ベースダイの仕様要求
SK hynix / Samsung / MicronDRAMダイ設計・製造、TSV・スタッキング、最終KGSDテスト
TSMCベースダイのN12FFC / N5 / N3P製造、IP提供(PHY、SerDes等)
GUC / Alchip / 世芯電子ASIC設計サービス、ベースダイのRTL設計・物理設計、PHY IP統合

確立事実 GUC(世芯電子)はTSMC子会社で、HBM4ベースダイのレイアウト設計でSK hynix・Micronの両方を支援している。AlchipはAMD MI400向けでHBM4ベースダイ設計に関与。推論 「ASIC設計サービス会社が、メモリ業界の新規参入障壁の鍵を握る」という構造が生まれており、これは半導体産業史上初めての現象。

TSMC側のキャパシティ問題

確立事実 NVIDIAはRubin世代でHBM4 × 8〜12スタック、Rubin Ultraで12〜16スタックを採用。1スタックあたりベースダイ~15mm²(N5)とすると、Rubin Ultra 1台で180〜240mm²のN5ウェハ消費が「メモリ側」だけで発生する。これはGPU本体ダイ(~800mm²)に対して20〜30%相当の追加N5キャパシティを要求し、TSMCは「アクセラレータ本体」「HBMベースダイ」「CPO ASIC」「DPU/NIC」のN5/N3P枠を顧客に割り当てる権限を実質的に握る。

推論 TSMC社内でHBMベースダイ専用のline assignmentが行われており、Rubin/MI400世代の出荷量はTSMCのN5/N3Pキャパシティ拡張ペースで実質的に律速される。これがメモリベンダ・GPUベンダ双方にとって新たな依存先となり、TSMCの価格交渉力が更に強まる。

マイルストーン

現在 (2026)20272028〜2030
HBM4ベースダイをTSMC N12FFC/N5で量産開始。PHY/コントローラ面積が15mm²(HBM3e 11mm²比+36%)。I/O電圧0.8V化で消費電力2倍効率化。SK hynix・Micron共にTSMCでベースダイ製造、GUC/Alchipが設計支援。C-HBM4E: 顧客仕様カスタムベースダイがN3Pへ移行。電圧0.75V・面積効率2.5倍。Vera Rubin Ultra・AMD MI500向けにco-design発注ピーク。Samsungが自社ファウンドリでベースダイ内製案検討。カスタムHBM市場が本格立ち上がり。メモリとロジックの境界が溶解。Near-Memory Compute機能(softmax/reduction/sparsity処理)がベースダイに常駐。HBM5世代でJEDEC仕様にカスタム要素が吸収されベースダイ「セミ標準化」(spec)。
