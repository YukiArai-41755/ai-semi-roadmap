HBF (高帯域フラッシュ)

High Bandwidth Flash — NAND-based stacked memory with HBM-class interface

ランプ/認定

大規模AI推論でHBM(高速・小容量・超高価)とNAND/SSD(低速・大容量・安価)の間に存在する10〜100倍の帯域・コスト断層を埋める新カテゴリ。NANDダイをHBMライクに垂直スタックし、HBMと同種のシリコンインターポーザインタフェース(2048bit/数百GB/s級)で演算チップに直接接続することで、容量はHBMの8〜16倍、コストはHBMの1/5〜1/10を狙う。読出し中心のKVキャッシュ、Llama 3 405B/MoE Mixture-of-Experts級の巨大モデル重み、Retrieval-Augmented Generation(RAG)のエンベディングインデックスなど、「容量重視・帯域は中位で良い」AIワークロードの受け皿として設計される。SanDisk(2025年Western Digitalから分離)とSK hynixの2025年8月標準化協業がアンカー、KioxiaとNVIDIAは「100M IOPS級SSD」で同じ問題に対する別解を提示。NAND陣営がHBM領域(SK/Samsung/Micronの牙城)へ攻め上がる楔であり、L4メモリ階層に「もう一段」を加える構造変化。

役割HBM容量を補完する大容量・準高帯域メモリ層
区分3D NAND積層型広帯域メモリ(新カテゴリ)
主配置CoWoSインターポーザまたはGPU近接、HBMと並列搭載

確立事実 HBMとSSDの間には帯域で~100倍、容量で~1000倍、$/GBで~100倍の連続的でない「空白」がある。HBFはこの空白を埋めるために設計される。推論 2028年のAI推論メモリBOMは「HBM4(or HBM4E)+ HBF + SOCAMM/LPDDR + SSD」の4層階層が標準化する可能性が高い。

技術原理 — NANDをHBMライクに使う仕組み

HBFは3D NAND(200層超のCharge Trap Flash)をDRAMライクのプロトコルでアクセスする。通常のSSDではNANDコントローラとの間にPCIeレイテンシが介在するが、HBFでは以下の工夫でレイテンシ・帯域を改善する。

技術要素狙い・実装
HBMライクI/Oインタフェース2048bit級ワイドI/O、シリコンインターポーザ直結、PCIeを介さない
並列バンク化NANDダイを多数バンクに分割、独立並行アクセスで実効帯域確保
SLC/MLC領域動的割当頻繁参照部はSLCモードで高速化、低頻度はTLC/QLCで容量効率
読出し最適化AI推論は読出し中心(書込みは初期ロードのみ)、NANDの読出し性能をフル活用
軽量コントローラ統合ベースダイ(ロジック)にECC・ウェアレベリング・キャッシュを内蔵
ECCの強化QLC NANDのビット誤り率(BER)~10-3を読出し時に~10-15級まで訂正

AI推論アプリケーションでの位置付け

HBFが最も効くのは「読出し中心・容量律速・帯域は中位で良い」ワークロード。

ユースケースHBFの役割
長コンテキスト推論(>1M token)のKVキャッシュHBM容量を超えたKVをHBFに退避、PagedAttention/vLLM拡張
MoE(Mixture-of-Experts)モデル非活性Expertの重みをHBFに置き、必要時にHBMへロード
巨大モデル(Llama 3 405B、GPT-5級)パラメータの一部をHBFに常駐、HBMに動的ロード
RAG(Retrieval-Augmented Generation)エンベディングインデックスを大容量HBFに保持
マルチテナント推論テナントごとのLoRAアダプタ重みをHBFにプール
